Remove /agora from the LingTai TUI

Prepared for Jason’s request to cancel the TUI /agora command while keeping the underlying recipe/import infrastructure intact.

What changed

  • Removed the /agora slash command from the command palette and command routing.
  • Deleted the old AgoraModel screen file (tui/internal/tui/agora.go).
  • Removed agora.* strings that only served the deleted command screen.
  • Removed /agora from bundled /help slash-command docs in English, Chinese, and Wen.
  • Updated repo/TUI anatomy docs so the deleted screen is no longer listed.
  • Added regression tests that guard against the command or help docs returning.

What intentionally stays

The change removes the TUI command/screen, not every historical or infrastructure reference to “agora”. The following remain by design:

  • First-run and recipe-import plumbing can still discover/import agora-style recipes.
  • Historical exported-network browser code remains in place for now, but it is no longer exposed by the removed /agora command.
  • Migration names and historical docs/blog posts remain untouched.
