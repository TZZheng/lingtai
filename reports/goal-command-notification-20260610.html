/goal command notification

结论：本 PR 给 TUI 增加 /goal 命令。它不会直接替 agent 创建 goal；而是向当前 agent 的 .notification/system.json 追加一个 source="goal.request" system event，让 agent 阅读 goal manual， 引导 human 明确 objective / criteria / reminder / cancellation semantics，确认后再由 agent 写入 .notification/goal.json。

为什么这样做

Jason 希望通过 /goal 触发一个 system notification：提醒 agent human 想设置 goal，要求 agent 先读 goal manual， 再 guide human to create a goal，并明确告知如何取消。TUI 只负责“发起引导”，不越过 agent/human 对话直接写死 goal。

实现概览

文件变化
tui/internal/tui/goal.go新增 writeGoalRequestNotification：创建/读取 .notification/system.json，保留已有 data.events，追加 goal.request，最多保留 20 条，temp-file + rename 原子写入。
tui/internal/tui/app.go/goal [draft] palette handler：检查当前 agent 存在且 alive，然后写入 system notification；失败时给本地 UI 提示。
tui/internal/tui/palette.go + tui/i18n/*.json把 /goal 加入 command palette，并补齐 EN/ZH/Wen 三语言文案。
lingtai-tui-help/assets/slash-commands.*.md三语言 /help 文档说明 /goal 是引导入口，不直接创建 goal 文件，并解释 reminder dismiss 与 cancel 的区别。
ANATOMY.md, tui/internal/tui/ANATOMY.md记录 /goal 是 TUI 对 .notification/ 的窄写入例外。

写入的 system event 语义

{
  "source": "goal.request",
  "ref_id": "goal.request:<timestamp>",
  "body": "Human wants to set or revise an active goal...",
  "at": "2026-06-10T...Z"
}

正文要求 agent：

  • 阅读 system-manual 下的 goal manual；
  • 和 human 明确 objective、success criteria、可选 reminder_delay_seconds 与 constraints；
  • 明确说明：取消 goal 需要删除 .notification/goal.json 或把 data.status 标记为 inactive/cancelled/done；
  • 明确说明：dismiss goal.reminder 只隐藏提醒，不取消 goal；
  • 在 human 确认细节前，不创建或覆盖 .notification/goal.json。
