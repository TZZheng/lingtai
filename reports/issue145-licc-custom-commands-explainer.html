Issue #145 — Document LICC/Telegram bridge and agent-level commands

Prepared for PR review, 2026-06-09.

Conclusion

Implemented as tutorial documentation. The communication lesson now explains how external MCP bridges such as Telegram translate messages into ordinary agent mailbox work via LICC, and how slash-like strings such as /poem or /status can be agent-level conversation conventions.

New teaching model

Telegram user message
  → Telegram Bot API
  → lingtai-telegram MCP bridge
  → LICC inbox event
  → LingTai kernel writes/wakes the agent mailbox
  → agent reads and replies on Telegram

Boundaries clarified

  • LICC delivers messages; it does not auto-execute kernel lifecycle actions.
  • Agent custom commands are normal agent behavior, not necessarily Telegram-native bot commands.
  • /refresh sent in chat is still text; it does not bypass authorization or permissions.
