Codex preset service tier option

PR explainer draft

Conclusion

This change makes the Codex preset editor match Codex's two practical modes:

  • normal: omit manifest.llm.service_tier.
  • fast: write manifest.llm.service_tier = "fast".

It deliberately does not default to flex, because local live verification showed flex parsed in the CLI but the current backend rejected it.

What changed

AreaChange
Preset editorAdds a Codex-only service_tier row rendered as normal / fast.
SerializationFor Codex, normal removes service_tier and fast stores "fast". Non-Codex providers keep any existing provider-specific service_tier.
Built-in Codex presetTested to omit service_tier by default.
i18nAdds labels in English, Chinese, and Wenyan locales.

Risk notes

  • The option is Codex-only and hidden for other providers.
  • For Codex, invalid values such as default or flex display as normal and are removed on commit.
  • For non-Codex providers, existing service_tier values are preserved because they may be provider-specific.
  • No global ~/.codex/config.toml is modified.
