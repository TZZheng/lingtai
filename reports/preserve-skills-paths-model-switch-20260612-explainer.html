Issue #311 · TUI preset editor · migration m037

Model switch no longer deletes skills.paths overrides

Switching the model in the preset editor used to wholesale-replace manifest.capabilities with the target model's defaults, silently deleting skills.paths and every other non-model-dependent capability override. Now only the genuinely model-conditional capabilities (web_search, vision) are reset. A one-time migration also repairs already-saved presets that were affected before the fix.

The bug

syncCapsToModel in tui/internal/tui/preset_editor.go ran on every model change (provider cycle and model cycle) and did:

m.working.Manifest["capabilities"] = defaultCapsFor(modelID)

defaultCapsFor only ever returns web_search (plus vision for image-capable models). So a preset with

"capabilities": {
  "web_search": {"provider": "zhipu"},
  "skills": {"paths": ["../.library_shared", "~/.lingtai-tui/utilities"]},
  "bash": {"yolo": true}
}

lost skills and bash entirely the moment the user cycled to a different model — even though those entries have nothing to do with which model is selected.

The fix

syncCapsToModel now mutates the existing capabilities map in place, iterating only over optionalCapabilities (web_search, vision):

caps := m.capsMap()
defaults := defaultCapsFor(modelID)
for _, capName := range optionalCapabilities {
    if def, ok := defaults[capName]; ok {
        caps[capName] = def
    } else {
        delete(caps, capName)
    }
}
  • web_search still resets to the target model's default backend.
  • vision still appears for vision-capable models and is removed for text-only ones.
  • Everything else — skills.paths, bash policy, any other core-capability kwargs — survives the switch untouched.
  • The existing guard is unchanged: free-text model ids not in the catalog (modelHasVision) leave capabilities completely alone.

Migration for already-affected saved presets

PR #312 also adds migration m037 preset-skills-paths. On next startup/upgrade, it scans global saved presets under ~/.lingtai-tui/presets/saved/ and silently restores the default skills path override when it is missing (including saved presets whose entire capabilities map is absent):

"skills": {
  "paths": ["../.library_shared", "~/.lingtai-tui/utilities"]
}
  • Idempotent: existing skills.paths values are preserved exactly.
  • Conservative: built-in templates are not patched because they are regenerated and already carry the default paths.
  • Durable: the preset is patched instead of individual agent init.json files, because active presets rematerialize agent capabilities on refresh/boot.
  • Migration etiquette: TUI migration version bumps from 36 to 37, and the portal registry receives the matching no-op version entry to keep the shared meta version space aligned.
