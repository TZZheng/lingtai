Issue #172 — Optional remote field for repo-backed skills

Prepared for PR review, 2026-06-09.

Conclusion

Implemented as a small catalog-metadata change. Skill entries now carry an optional configured Git remote URL when their SKILL.md directory is inside a Git worktree with a configured remote. Non-repo or no-remote skills continue to omit the value.

What changed

  • Added Remote to the internal skillEntry and MarkdownEntry catalog structs.
  • Added gitRemoteForDir: prefer origin; otherwise choose a deterministic first remote; otherwise return empty.
  • Propagated the optional value through both agent-scoped and legacy library catalog builders.

Scope boundaries

  • No full provenance/evaluation schema.
  • No skill loading behavior changes.
  • No PR/issue/test metadata.
  • No visible UI badge yet; this PR exposes catalog metadata for follow-up UI use.

Validation

  • go test ./internal/tui/ ./i18n/ -run 'SkillRemote|Library|Markdown|Scan' -v passed.
