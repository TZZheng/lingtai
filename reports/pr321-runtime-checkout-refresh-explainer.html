Runtime Checkout Refresh Documentation

Explainer for docs-runtime-checkout-refresh-20260612

TL;DR

This docs-only change adds the missing developer workflow Jason requested: after a kernel PR is merged, verify the actual runtime import paths, fast-forward the editable checkout if stale, call system(refresh), then rerun the probe before declaring the fix live.

Baseline

The existing lingtai-dev-guide explained how to install the kernel editable into ~/.lingtai-tui/runtime/venv, but it did not cover the production pitfall we just hit: a running agent may import from an editable checkout that is behind origin/main, a detached worktree, or a legacy addon checkout.

Risks and decisions

  • This is docs-only; no runtime behavior changes.
  • The fast-forward recipe explicitly says git switch main only if safe, to avoid trampling a worktree branch.
  • The probe includes both curated paths and compatibility wrappers so developers notice old standalone addon repos.
