Issue #100 — Anti-pattern: text-consistency drift in iterative empirical paper writing

Docs/skill change Branch: fix/academic-text-data-drift Source: tui/internal/preset/skills/swiss-knife/reference/academic-research/

TL;DR

When an empirical paper is written iteratively with reviewer agents, the prose can become steadily more polished and internally consistent while quietly drifting away from the data on disk. Reviewer agreement validates text consistency, not data correspondence — so the draft can read cleanly, pass every reviewer round, and still misdescribe what the experiments actually did.

This change teaches the bundled academic-research skill to recognize and prevent that failure mode: a new anti-pattern reference, a pre-writing data-anchoring guard in the LaTeX pipeline, and routing from the skill's index and Known caveats.

Scope: documentation/skill content only. No Go runtime behavior changes; one bundling test gains an assertion for the new file.

Baseline (the incident)

During iterative empirical-NLP paper writing, a writer agent plus reviewer agents restructured the experimental framing across 9+ rounds without re-verifying claims against the data on disk. The text grew more internally consistent and more polished — but wrong.

The draft described "Experiment 2" as a single-agent-with-tools setup. The actual Exp 2 data lived in three sibling directories (a chat/tool/replaced split, 21 model pairs, no tools). Every reviewer round confirmed the draft was consistent — and described an experiment that was never run.

Root cause: each editing round optimized the text against itself rather than against the artifacts that produced it. Reviewer consensus was treated as evidence the claims were correct, when it only ever measured whether the prose hung together.

What changed

1. New anti-pattern reference

reference/anti-pattern-text-consistency-vs-data-correspondence.md — an actionable guide covering: the trigger pattern (describing experiments from memory/old drafts, fixing "confusing" feedback by rewriting prose, reframing sections without re-opening data, treating reviewer agreement as correctness); what to do instead (establish ground-truth correspondence first — list data, inspect a real result file, read the runner code, keep a claim → evidence map, re-derive on every reframe); why review agents don't catch it; an explicit anti-fix (don't rewrite for internal consistency); a detection checklist; and Related links including the multi-agent echo-chamber family.

2. LaTeX pipeline guard

reference/pipeline-latex-writing.md gains a "Before you write" block at the top — anchor prose to data before drafting and on every reframe — plus a See Also cross-reference to the new anti-pattern.

3. Skill index + caveats routing

SKILL.md gains an escape-hatch quick-path ("My empirical draft keeps getting reframed / reviewers 'agree'"), a Standalone references entry, and a Known caveat bullet that routes paper-writing drift to the new file.
