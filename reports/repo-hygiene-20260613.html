LingTai main repo hygiene clarification

Scope: conservative cleanup PR for Lingtai-AI/lingtai; generated after Claude proposal and conservative synthesis.

TL;DR

This PR intentionally does not delete historical reports, discussions, or prompt archives. It makes the repository hygiene contract explicit and adds artifacts/ as the local-only output directory for throwaway generated files.

Baseline

  • Claude Code proposed removing several generated-looking reports/*.html files and adding broad ignore patterns.
  • Local evidence showed reports/ is part of the current PR/release explainer workflow: the dev guide requires human-facing HTML explainers under reports/.
  • discussions/ holds agent-filed patch/design specs and should not be bulk-deleted during generic cleanup.
  • prompt/archive/ has historical prompt material and references from older planning docs; it remains manual-review territory.
  • tui/bin/ and portal/bin/ are already ignored by their package-level .gitignore files, so no root change is needed for them.

Risks / decisions

  • No source, build, runtime, or release behavior changes.
  • No deletion of tracked historical artifacts in this PR.
  • Future cleanup of individual reports, discussions, or prompt archive files should be owner-approved and issue-specific.
